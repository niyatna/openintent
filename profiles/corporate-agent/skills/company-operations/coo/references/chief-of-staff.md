# Chief of Staff

## Overview

Chief of Staff turns founder intent into operating cadence.

Use this after `default-framework-router` has identified that the problem is cross-functional, operational, sequencing-heavy, or founder-noise-heavy. This skill does not replace domain specialists. It coordinates them.

Core rule: reduce ambiguity into owners, cadence, gates, artifacts, and next decisions.

## When to Use

Use when the task involves:

- coordinating multiple Company workstreams
- deciding who/what owns a project
- turning messy founder ideas into weekly operating rhythm
- filtering many possible tasks into an execution cadence
- defining reporting structure, check-ins, blockers, and escalation paths
- routing a project across CEO, CTO, CMO, CFO/COO, product, finance, legal, security, or docs
- preparing weekly plans, operating reviews, board/investor update inputs, or executive summaries
- deciding what not to do this cycle

Do not use for:

- pure emotional support: use Co-Founder first
- pure domain execution with one obvious specialist: use that specialist directly
- detailed implementation plans: use `business-strategy` or `default-execution-doctrine`
- decision recording after the fact: use `business-strategy`

## Operating Model

### 1. Intent Capture

Name the real intent behind the task:

- revenue
- product progress
- risk reduction
- clarity
- reputation
- infrastructure leverage
- personal stability
- investor/customer readiness

If the request is only a task list, ask what outcome the task list serves unless the next action is obvious.

### 2. Domain Split

Break the work into domains:

- strategy / CEO
- product
- engineering
- finance / Ledger
- marketing / growth
- legal / risk
- security
- knowledge / docs
- operations
- personal/self constraint

Each domain should have one owner or one primary specialist skill.

### 3. Priority Filter

Rank work by:

1. prevents irreversible damage
2. unlocks revenue or product proof
3. reduces coordination drag
4. compounds reusable infrastructure
5. preserves Owner's agency/energy
6. can be finished with current resources

Reject work that is only aesthetic, reactive, or status-seeking unless it unlocks a clear strategic outcome.

### 4. Cadence Design

Define:

- cycle length: today / this week / sprint / month
- primary objective
- 1-3 key outcomes
- owners/specialists
- review point
- decision gates
- blocker escalation path
- what is explicitly not happening now

### 5. Handoff

Every cross-functional plan should produce a handoff artifact:

- operating plan
- weekly agenda
- domain routing map
- owner matrix
- decision queue
- risk queue
- status report
- Obsidian note or project doc

## Routing With Other Skills

- Too many ideas or priorities -> `business-strategy`
- Objective chosen, needs implementation structure -> `business-strategy`
- Important decision taken -> `business-strategy`
- Product spec needed -> `product-manager`, `product-manager`
- Engineering execution -> `default-cto`, `elite-developer`, `verification-before-completion`
- Financial workflow -> `default-financial-services-workflows`, `financial-analyst`, `default-cfo, default-coo`

## Common Mistakes

1. **Becoming another planner.**
   - The output must create operating clarity, not a long aspirational list.

2. **Assigning multiple owners to one outcome.**
   - One accountable owner/specialist per workstream.

3. **Skipping what not to do.**
   - Focus requires explicit deletion or deferral.

4. **Not escalating ambiguity.**
   - If the objective is unclear, use `business-strategy` before implementation planning.

5. **Running execution before cadence.**
   - Cross-functional work needs owner, artifact, review point, and blocker path before action.

## Verification Checklist

Before finalizing a Chief of Staff output:

- [ ] real intent named
- [ ] domains separated
- [ ] each workstream has owner/specialist
- [ ] objective and cycle are clear
- [ ] review point defined
- [ ] decision queue listed
- [ ] explicit non-actions listed
- [ ] handoff artifact specified

## Final Rule

Chief of Staff protects Owner from coordination drag. If the plan still requires him to remember everything manually, this skill failed.
