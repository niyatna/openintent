# Personal Brand / Identity Vault Population Workflow

Use this reference when Galih asks to create or fill a durable personal-brand, founder-voice, or identity-strategy vault area from known context, especially prompts like `isi semuanya berdasarkan yg lu tau tentang gua`.

## Trigger

- User provides an Obsidian folder/tree outline and expects it to exist.
- User asks to fill many notes from known context, memory, Hindsight, or canon.
- The output is a durable personal/founder/brand/strategy knowledge base, not a one-off chat answer.

## Workflow

1. **Route as identity + galyarder-ceo + Obsidian/file work.** Use this skill for the content architecture; use the Obsidian skill for filesystem operations.
2. **Ground before writing.** Use hot context plus available Hindsight/memory and existing Obsidian canon/libraries. For Galih personal brand work, useful sources include core memory, writing/taste, brand/design, life timeline/private human model when appropriate, active project map, and Galyarder Labs canon.
3. **Separate evidence levels.** In notes, distinguish `Known`, `Inferred`, and `Assumed` when claims could be stale or partially inferred.
4. **Use privacy and source metadata.** Add frontmatter such as `status`, `privacy`, `source_basis`, and `last_filled` for durable strategic notes.
5. **Create navigable structure.** Build parent index notes and child notes with Obsidian wikilinks. Use real content, not empty placeholders, when the user asks to fill everything.
6. **Protect private context.** Private memories can inform judgement, but do not turn romantic history, childhood pain, family conflict, or raw emotional anchors into public-facing brand content unless Galih explicitly asks. Convert them into public-safe doctrine: agency, systems, proof, integrity, human core, responsibility, and operating lessons.
7. **Label unverified market material.** Competitor/market claims from memory or general category knowledge should be marked as working assumptions or research gaps until live research validates them.
8. **Verify before claiming done.** Check expected file count, no leftover placeholder text, no accidental `TODO/TBD`, and sample-read 2–3 important notes.

## Completion Evidence Shape

Report briefly:

- target path;
- number of markdown files written/updated;
- placeholder/TODO scan result;
- sample notes verified;
- one-line note on source grounding.

## Pitfalls

- Do not stop after creating only the folder skeleton when the user expected populated notes.
- Do not ask broad discovery questions if the user explicitly says to use what is already known and grounding sources are available.
- Do not fabricate market facts or competitor details; label unknowns clearly.
- Do not dump private memories directly into public brand copy. Use them as internal source material only.
